# 毛沢東.skill - クイックスタートガイド (日本語)

## 🎯 はじめに

毛沢東.skillは、毛沢東の方法論と人格スタイルに基づくAIアシスタントで、現代的な6層認知アーキテクチャを採用しています。以下の機能を提供します：

- 毛沢東の6つの核心的方法論を使用したインテリジェント分析
- 4段階の漸進的学習システム
- 関係性探索を含む674の概念ライブラリ
- 毛沢東の言語スタイルと思考パターン

## 🏗️ 6層認知アーキテクチャ

毛沢東.skillは現代的な6層認知アーキテクチャを採用しています：

1. **ユーザーインターフェース層** - コマンド解析とレスポンスフォーマット
2. **分析意思決定層** - インテリジェント推薦（78%以上の精度）
3. **メソッド実行層** - 6つの具体的な分析方法
4. **知識検索層** - 674の概念と4段階学習パス
5. **パフォーマンス監視層** - リアルタイム監視とキャッシュ最適化
6. **統合インターフェース層** - 統一インターフェースと後方互換性

**バージョン**: 1.2.0 (6層アーキテクチャ版)
**リリース日**: 2026年4月9日

## 🚀 クイックスタート

### 1. インストールと設定

#### 完全プロジェクトインストール（推奨）
```bash
# OpenClawユーザー
git clone https://github.com/wwwaapplleecu-source/mao-skill ~/.openclaw/workspace/skills/mao-colleague

# Claude Codeユーザー
git clone https://github.com/wwwaapplleecu-source/mao-skill .claude/skills/mao-colleague

# Python依存関係のインストール（必要な場合）
pip install -r requirements.txt
```

> **📦 プロジェクト即スキル**:  `mao-colleague`プロジェクト全体が完全な毛沢東.skillです。一度インストールするだけで全ての機能をすぐに使用できます。

### 2. コアコマンドアーキテクチャ

毛沢東.skillは統一されたメインコマンド＋サブコマンドアーキテクチャを使用します：

```bash
# ショートカット（インテリジェント分析）
/mao [質問]

# 完全コマンドアーキテクチャ
/mao help                      # ヘルプ取得（インテリジェントガイド）
/mao analyze [質問]           # インテリジェント分析（--methodパラメータ対応）
/mao learn                     # 学習システム（4段階漸進パス）
/mao concepts                  # 概念クエリシステム（674+概念）
/mao compare                   # 方法比較システム
/mao settings                  # パーソナライズ設定システム
```

### 3. インテリジェント分析例

#### インテリジェント推薦
```bash
/mao 会社部門間の協力矛盾を分析

# システムが最適な方法を自動推薦（例：矛盾分析法、78%信頼度）
```

#### 指定方法分析
```bash
/mao analyze --method=contradiction プロジェクトの主要矛盾を特定
/mao analyze --method=practice ワークフローを改善する方法
/mao analyze --method=investigation ユーザー調査計画を設計
```

#### 学習開始
```bash
/mao learn                     # 学習開始（インテリジェントパス推薦）
/mao learn --path=beginner     # 入門パス選択（15分）
/mao learn --path=basic        # 基礎パス選択（1時間）
/mao learn --path=advanced     # 応用パス選択（3時間）
/mao learn --path=professional # 専門パス選択（10時間）
```

#### 概念クエリ
```bash
/mao concepts                  # 核心概念リスト表示（674+概念）
/mao concepts contradiction    # 「矛盾」概念の詳細説明表示
/mao concepts --search=mass    # 「大衆」を含む関連概念検索
```

## 📖 使用例

### 例1：インテリジェント矛盾分析
```
ユーザー: /mao チーム協力矛盾を分析

毛沢東.skill: まず主要矛盾を見つけ出します。チーム協力における主要矛盾は個人目標と集団目標の矛盾であり、副次矛盾にはコミュニケーション不足、資源配分不均等、役割定義不明瞭などがあります。矛盾を有利な方向へ転化させるためには、共通目標を確立し、コミュニケーション調整を強化し、役割分担を明確にする必要があります。
```

### 例2：実践論方法指導
```
ユーザー: /mao analyze --method=practice 製品設計を改善する方法

毛沢東.skill: 製品設計を改善するには実践-認識循環に従います。まず小範囲で設計実践を行い、感性的認識を獲得します；実践経験をまとめ、理性的認識を形成します；理性的認識を用いてより大規模な設計実践を指導します；実践-認識-再実践循環を通じて、設計を徐々に完成させます。
```

### 例3：学習システムガイダンス
```
ユーザー: /mao learn --path=beginner

毛沢東.skill: 毛沢東方法論学習へようこそ！現在のパス：入門パス（15分）。学習目標：毛沢東方法論の基本概念と応用価値を理解する。モジュール進捗：0/3（0%）。第一モジュール：毛沢東方法論概要。学習目標：毛沢東方法論に対する全体的理解を確立する。
```

## 🎭 毛沢東.skill出力スタイルの理解

### 1. 方法論的特徴
- **矛盾分析法**: 主要矛盾と副次矛盾を区別、矛盾転化を分析
- **実践論思考**: 実践-認識-再実践-再認識循環を強調
- **調査研究法**: 調査なくして発言権なし、典型と普遍を結合
- **大衆路線法**: 大衆の中から、大衆の中へ
- **戦略的思考**: 戦略的に軽視、戦術的に重視、持久戦思考
- **総合分析法**: 最適な分析方法をインテリジェントに選択

### 2. 言語スタイル特徴
- **同志たち始まり**: よく「同志たち、」で始まる
- **並列構文**: 「…も…も…も…も」等の並列パターン使用
- **古典的比喩**: 「紙の虎」、「麻雀を解剖する」等の毛沢東古典比喩使用
- **革命用語**: 「革命」、「闘争」、「勝利」等の用語使用
- **要約昇華**: 分析後に要約と行動提言を提供

### 3. 現代的特徴
- **インテリジェント推薦**: 質問内容に基づき最適な分析方法を自動推薦
- **学習システム**: 4段階漸進学習パス、個別化進捗追跡
- **概念ネットワーク**: 674概念関係ネットワーククエリ
- **パフォーマンス監視**: 内蔵パフォーマンス監視とキャッシュ最適化
- **後方互換性**: 全ての旧コマンドを引き続きサポート

## 🔧 高度な使用テクニック

### 1. 問題提起最適化
- **具体化**: 「どうすればよいか」ではなく「具体的にどのように実施するか」と尋ねる
