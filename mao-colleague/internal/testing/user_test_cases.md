# 毛泽东.skill 用户测试用例

## 🎯 测试目标
验证新命令架构（模块三）的可用性、易用性和功能完整性。

## 📋 测试用例列表

### 测试用例 1: 智能快捷命令
**测试命令**: `/mao 分析公司部门协作矛盾`
**预期结果**:
- 系统识别问题类型为"分析"
- 智能推荐合适的方法（应为"矛盾分析"或"综合分析"）
- 提供结构化分析步骤
- 包含毛泽东核心概念和语录

**验证点**:
- [ ] 是否正确识别问题类型
- [ ] 推荐方法是否合理（置信度>50%）
- [ ] 响应结构是否清晰
- [ ] 是否包含相关概念和语录

### 测试用例 2: 指定分析方法
**测试命令**: `/mao analyze --method=矛盾 分析团队冲突`
**预期结果**:
- 使用矛盾分析方法
- 提供矛盾分析四步法
- 包含主要矛盾、次要矛盾等核心概念

**验证点**:
- [ ] 是否正确使用指定方法
- [ ] 分析步骤是否完整
- [ ] 概念解释是否准确

### 测试用例 3: 纯方法论风格
**测试命令**: `/mao analyze --style=work 制定项目计划`
**预期结果**:
- 响应为纯方法论风格（不包含毛泽东人格风格）
- 使用战略思维方法
- 提供战略规划步骤

**验证点**:
- [ ] 是否排除人格风格内容
- [ ] 是否使用合适的方法论
- [ ] 响应是否专业、实用

### 测试用例 4: 学习系统介绍
**测试命令**: `/mao learn`
**预期结果**:
- 显示四级学习路径概览
- 提供快速开始指南
- 展示核心方法论列表

**验证点**:
- [ ] 学习路径介绍是否清晰
- [ ] 导航建议是否明确
- [ ] 界面是否友好

### 测试用例 5: 学习路径详情
**测试命令**: `/mao learn --path=入门`
**预期结果**:
- 显示入门路径详细信息
- 包含学习目标、时长、目标人群
- 列出所有学习模块

**验证点**:
- [ ] 路径信息是否完整
- [ ] 模块描述是否清晰
- [ ] 开始学习指引是否明确

### 测试用例 6: 学习进度查看
**测试命令**: `/mao learn progress`
**预期结果**:
- 显示当前学习进度概览
- 展示各路径进度条
- 提供学习推荐

**验证点**:
- [ ] 进度显示是否直观
- [ ] 进度条是否准确
- [ ] 推荐是否合理

### 测试用例 7: 开始学习路径
**测试命令**: `/mao learn start --path=入门`
**预期结果**:
- 开始入门学习路径
- 显示第一个模块内容
- 提供下一步指引

**验证点**:
- [ ] 是否能成功开始路径
- [ ] 模块内容是否完整
- [ ] 指引是否清晰

### 测试用例 8: 专题学习
**测试命令**: `/mao learn 矛盾论`
**预期结果**:
- 显示矛盾论专题内容
- 包含理论、应用、核心概念
- 提供前置要求和相关命令

**验证点**:
- [ ] 专题内容是否完整
- [ ] 结构是否清晰
- [ ] 学习路径指引是否合理

### 测试用例 9: 概念查询
**测试命令**: `/mao concepts 矛盾`
**预期结果**:
- 显示矛盾概念的详细解释
- 包含定义、分类、应用
- 提供相关概念链接

**验证点**:
- [ ] 概念解释是否准确
- [ ] 信息是否全面
- [ ] 导航是否方便

### 测试用例 10: 向后兼容测试
**测试命令**: `/mao-analyze 矛盾 公司内部问题`
**预期结果**:
- 识别为老命令，提供迁移建议
- 使用矛盾分析方法分析问题
- 保持原有功能不变

**验证点**:
- [ ] 是否识别老命令
- [ ] 迁移建议是否友好
- [ ] 功能是否保持兼容
