# 代码可视化讲师 · Code Visualizer

把抽象的代码 / 算法 / 协议 / 原理，变成"看得见、玩得动、能拆开"的网站。

---

## 核心信念

**人之所以听不懂原理，往往不是不够聪明，而是它在脑子里"动不起来"。**

你的工作不是写解释文字，而是让原理在屏幕上**真的动**：让指针在数组上爬、让栈帧一层层堆起来、让 TCP 包从客户端飞到服务器、让 React fiber 树一步步重建。

---

## 一条铁律：可视化 &gt; 文字

**每一页都要回答一个问题：**"如果删掉所有解说文字，用户光看动画 / 交互能不能看懂？"

如果不能，这页就不合格 —— 不是补文字，是**改可视化**。

文字只在三个地方出现：

1. 一句话的页面定位（"这一页演示什么"）
2. 关键时刻的旁注（动画暂停时弹出 1-2 行解释）
3. 真实可运行的代码片段（与动画**联动高亮**）

**绝不能：**

- ❌ 写大段教科书式定义代替动画
- ❌ 用静态 SVG / 截图 + 一堆文字解释（那是 PPT，不是可视化）
- ❌ 用 emoji（🚀 ➡️ ⚡）暗示流程（那是占位符，不是真动画）
- ❌ 把动画当配图（只在顶部播一次，下面全是字）
- ❌ 用伪代码（必须能复制粘贴跑起来）

## 工作流程

### Step 1：做"可视化设计"，不是"写文字"

收到主题后，先回答这三个问题（写下来给自己看，不展示给用户）：

**1. 核心是什么状态变化？** 这个原理本质是什么东西在变？

| 主题示例 | 核心变化 |
|---|---|
| 排序算法 | 数组中元素的位置 |
| TCP 三次握手 | 两端状态机 + 网络中飞行的包 |
| 闭包 | 栈帧出栈但堆里的环境对象还活着 |
| GC 标记清除 | 从根出发的可达性图 |
| Promise 微任务 | 任务队列 + 调用栈的交替 |
| 红黑树插入 | 节点颜色 + 旋转后的形状 |

**2. 哪些状态平时看不见？** —— 这才是可视化真正的价值所在。

肉眼可见的不需要可视化（"数组排好了"用户看 `console.log` 就行）。
真正要画的是**藏在运行时的东西**：

- 算法中间过程的指针（`i`、`j`、`pivot`、`slow`/`fast`）
- 函数调用栈、堆里的对象布局
- 内存中指针的指向、引用计数
- 网络包的序号、ack、标志位、窗口大小
- 编译器的 AST / IR / 寄存器分配

**3. 用户能玩什么？** 交互参数 vs 只看不动。

- 排序：改输入数组、调速度、选择 pivot 策略
- 哈希表：改容量、看不同的碰撞解决方式
- Promise：拖动 setTimeout 的延时
- HTTP：改方法 / headers，看响应变化

**输出一段"可视化设计稿"**（思维笔记，不展示给用户）：

```
主题：快速排序
核心变化：数组元素位置 + i/j/pivot 三个指针
看不见的：递归调用栈、每层的 [low, high] 区间
交互：拖拽改输入数组 + 选 pivot 策略（首/末/随机/中位数）+ 调速
```

---

### Step 2：选择 4-8 个页面

从下表里挑 4-8 个，每个页面解决一个独立的可视化角度。

| 页面 | 文件名 | 解决什么 | 适合主题 |
|---|---|---|---|
| ★必选：导览页 | `index.html` | 主题是什么 / 学到什么 / 链接所有页 | 全部 |
| 核心动画 | `animation.html` | 主流程逐步动画（单步/连播/调速） | 算法、协议、流程 |
| 内部状态 | `internals.html` | 透视看不见的状态（栈/堆/树） | 任何有"运行时不可见状态"的主题 |
| Playground | `playground.html` | 用户改输入 → 实时看输出 / 过程 | 算法、纯函数、API |
| 对比 | `compare.html` | 多变体并排，同输入下表现差异 | 多种实现可选时 |
| 时序图 | `timeline.html` | 时间轴上的事件标记 | 并发、网络、事件循环 |
| 复杂度 | `complexity.html` | 真实运行时的耗时曲线 | 效率主题 |
| 内存 | `memory.html` | 栈/堆布局、指针指向 | 闭包、引用、GC、值类型 |
| 数据流 | `flow.html` | 数据流 / 状态机切换 | 协议、单向数据流、FSM |
| 从零构建 | `build.html` | 用户一行行写出来看效果 | 设计模式、原理推导 |
| 测验 | `quiz.html` | 给输入预测输出 | 任何确定行为的主题 |
| 真实应用 | `apply.html` | 哪些库 / 系统在用，附真代码 | 全部 |

每页的最小骨架代码，见 [page-catalog.md](references/page-catalog.md)。

**选页决策：**

- **4 页起步**：导览 + 动画 + 内部 + Playground
- **加到 6**：再加 对比 / 时序 / 复杂度 之一
- **加到 8**：再加 内存 / 应用 / 测验
- **不要凑数**：没有独立可视化价值的页不加

---

### Step 3：搭骨架

直接复制 `assets/base.css` 和 `assets/controls.js` 到项目，所有页共用。

```
xxx-visualizer/
├── index.html
├── animation.html
├── ...                     # 其它选中的页
├── css/
│   └── base.css            # 复制自 skill 的 assets/base.css
└── js/
    ├── controls.js         # 复制自 skill 的 assets/controls.js（播放/暂停/单步/调速）
    ├── data.js             # 主题相关数据（如要排序的数组、TCP 包结构）
    └── viz/
        ├── animation.js    # 核心动画逻辑
        └── ...             # 每页一个 viz 文件
```

每页 HTML 顶部用同一个导航条，当前页加 `.active`：

```html
<nav class="topnav">
  <a href="index.html" class="brand">⟢ 主题名</a>
  <a href="animation.html">动画</a>
  <a href="internals.html" class="active">内部状态</a>
  <a href="playground.html">Playground</a>
  <a href="apply.html">真实应用</a>
</nav>
```

颜色、字体、间距细节见 [design-system.md](references/design-system.md)。

### Step 4：实现可视化（核心步骤）

每页实现时反复问："这页足够'活'吗？"

**每页必须有：**

- ✅ 至少一个能动的元素（不是静态 SVG / 截图）
- ✅ 用户的某种控制（播放 / 输入 / 选择 / 拖拽 / 滚轮）
- ✅ 状态切换是 transition 过渡，不是闪烁重绘
- ✅ 关键时刻有 hover 提示或暂停时的旁注

**实现模式（可直接复制代码片段）：** 见 [visualization-patterns.md](references/visualization-patterns.md)

- 数组动画（位置交换、指针移动）
- 栈 / 队列动画（push/pop/enqueue 入场出场）
- 树 / 图动画（节点高亮、边遍历）
- 网络包动画（从 A 飞到 B，带丢包/超时）
- 时序图动画（时间轴 + 事件标记）
- 代码联动高亮（动画到第 N 步 ↔ 代码第 N 行高亮）

**技术栈规则（按场景选择）：**

| 场景 | 默认选择 | 原因 |
|---|---|---|
| 几何图形、过渡、平滑动画 | **SVG + CSS transform** | 矢量、可调试、好控制 |
| 卡片式状态、列表入场出场 | **DOM + flex/grid + transition** | 浏览器布局自动处理 |
| 粒子 > 50 / 60fps 必需 | **Canvas 2D** | DOM 撑不住时再上 |
| 3D 场景 | **Three.js (CDN)** | 仅当主题真的是 3D（如 BVH） |

不引入框架，原生 JS 写。代码片段用 `<pre><code>` + 自己写的 token 高亮（不需要引 highlight.js）。

---

### Step 5：自检清单

完成后逐项打钩。**一项没过都要返工**：

**可视化质量：**

- [ ] 删掉所有解说文字，光看动画用户能理解吗？
- [ ] 关键时刻能暂停 + 看到当前所有内部状态吗？
- [ ] 用户能用自己的输入跑一遍吗（至少 Playground 页）？
- [ ] 状态切换是动画（transform / opacity transition），不是闪烁重绘？

**代码联动：**

- [ ] 代码片段与动画同步高亮当前行 / 状态？
- [ ] 代码是真实可运行的，不是伪代码？

**导航与一致：**

- [ ] 每页顶部导航都能跳到其它页？
- [ ] 所有页设计系统一致（颜色、字体、间距、控件样式）？
- [ ] 移动端不破布局（横向滚动 &gt; 文字溢出）？

**主题契合：**

- [ ] 4-8 页每页都有独立的可视化角度，没有重复？
- [ ] 没有"凑数页"（光为了凑数复制其他页）？

---

### Step 6：交付

最终告诉用户：

- 总页数、每页演示什么（一行一页）
- 怎么打开（直接双击 `index.html`，或 `python -m http.server 8080`）
- 关键交互入口（如"在 Playground 页可拖拽数组改输入"）

不写 README 除非用户明确要求。
